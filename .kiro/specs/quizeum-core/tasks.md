# Implementation Plan

## 1. Foundation: 基盤インフラおよび共通定義

- [ ] 1.1 ドメインデータモデルの拡張定義
  - カバー画像、問題アタッチ画像、制限時間（タイマー）、短答式（文字入力）問題の表記揺れ許容リスト、面白さ評価、体感難易度投票、指摘レポート、通報ドキュメント、および称号バッジなどの型定義。
  - `src/types/index.ts` が TypeScript エラーなくコンパイルされ、拡張された型が完全に定義されていること。
  - _Requirements: 2.6, 2.7, 2.8, 2.10, 3.6, 4.5, 8.3, 9.3, 11.2, 12.1_

- [ ] 1.2 コモンデータストアコンバーターおよびモデレーションフィルタ設定
  - Firestore への読み書き時の自動変換設定、および不適切ワード（NGワード）自動チェックロジックの実装。
  - 不適切な単語をリストに追加し、入力時に関連ロジックが正しくNG判定（チェック）を実行して保存をブロックできること。
  - _Requirements: 11.1_

- [ ] 1.3* テストインフラストラクチャのセットアップ
  - クイズ、解答履歴、ソーシャル、およびモデレーション各サービスの単体テスト・結合テスト実行環境。
  - テストコマンドにより、新規追加のサービスに対するテストが実行可能であり、すべてパスすること。
  - _Requirements: 3.4, 8.3, 11.1_

## 2. Core: 各種ドメインサービスの開発

- [ ] 2.1 (P) クイズ作成・投稿・管理サービスの構築
  - クイズの新規作成、更新、下書き保存、削除、および公開完全性検証（空欄、選択肢、正解が1つ以上あるかのバリデーション）。
  - 空の問題や正解未選択の問題があるクイズを公開しようとした際、エラーを返し公開をブロックできること。
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.9_
  - _Boundary: QuizService_

- [ ] 2.2 (P) プレイ結果の履歴保存および復習クイズ抽出サービスの構築
  - プレイヤーの解答結果の永続化、および過去に間違えた問題（弱点克服）の履歴からの自動抽出。
  - 保存された挑戦履歴から、過去に間違えた問題IDのみが順序を保って正確に抽出されること。
  - _Requirements: 3.4, 10.3_
  - _Boundary: AttemptService_

- [ ] 2.3 (P) ソーシャルインタラクションおよびレーティング・難易度投票サービスの構築
  - ブックマーク追加/解除時のアトミックなカウンタ更新、プレイ完了後の面白さ評価（星1〜5）および体感難易度投票のアトミック集計。
  - ユーザーが評価・投票を送信した際、クイズドキュメントの平均星評価および各難易度カウンターがアトミックに加減算されること。
  - _Requirements: 4.2, 4.3, 4.5, 4.6_
  - _Boundary: SocialService_

- [ ] 2.4 (P) クローズド間違い・別解指摘レポートおよび通知システムの構築
  - 作家へのクローズド指摘レポート送信、作家向け指摘一覧の取得、指摘の解決、および指摘ユーザーへの「修正完了」通知生成。
  - 作家が指摘された問題を修正して「解決」ステータスにした際、指摘者への通知リストに修正完了メッセージが自動で挿入されること。
  - _Requirements: 12.1, 12.2, 12.3_
  - _Boundary: ReportService_

- [ ] 2.5 (P) ユーザー通報および自動保留モデレーション機能の構築
  - 不適切コンテンツの通報処理、および通報件数が一定数（しきい値）に達した際に自動でクイズを非公開（保留）にするトランザクション。
  - 同一クイズに対し5件以上の異なるユーザーから通報が送信された時点で、クイズの `isPublished` が自動的に `false`（非公開）に変更され、検索結果から即時に排除されること。
  - _Requirements: 11.2, 11.3_
  - _Boundary: ModerationService_

## 3. Integration: UIコンポーネントおよび画面の構築・統合

- [ ] 3.1 高度な検索・探索フィルタダッシュボード（ホーム画面）の統合
  - 部分一致、複合フィルタ（ジャンル、難易度、問題数）、サジェスト機能、および未プレイ/プレイ済みの絞り込みオプション。
  - ホーム画面で「ジャンル:歴史」かつ「難易度:普通」かつ「未プレイ」を選択した際、対象となる公開クイズのみが正確にフィルタリングされて表示されること。
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5_

- [ ] 3.2 Server Componentsベースのクイズ詳細画面とSEO動的メタデータ生成の統合
  - 静的・サーバーレンダリングHTMLの即時返却、動的なOGP（タイトル、説明、カバー画像）メタデータ注入。
  - クローラーからのリクエスト（JavaScript無効状態）でクイズ詳細ページをリクエストした際、HTMLヘッドタグ内にクイズタイトルとOGPメタデータが正しく埋め込まれて返却されること。
  - _Requirements: 7.1, 7.2, 7.4_
  - _Depends: 2.1_

- [ ] 3.3 状態保護タイマーおよび多様なモードを備えたクイズプレイ画面の統合
  - 通常プレイ（1問ごとに正誤＆解説）、模擬試験（全問解いて一括採点）、フラッシュカード（暗記モード）、出題順シャッフル。
  - プレイ中リロード時の解答セッションの保護（localStorageを用いた状態復旧）、段階的なヒント表示、マークダウン詳細解説のレンダリング。
  - 短答式（文字入力）問題における表記揺れの自動許容判定（ひらがな、カタカナ、英字の違いなどの許容）。
  - 通常モードでプレイ中にページをリロードした際、現在の進行状況（問数、正答スコア）を維持したまま、解答画面が再開されること。
  - _Requirements: 2.6, 2.10, 3.1, 3.2, 3.3, 3.5, 3.6, 3.7, 3.8, 3.9, 3.10, 8.3, 8.4, 10.1, 10.2, 10.4_
  - _Depends: 2.2_

- [ ] 3.4 作家ダッシュボードとアナリティクス表示の統合
  - プレイ数、評価、難易度投票のビジュアル表示。問題別の選択肢の選択比率（正解・誤答分布）のグラフ化。称号バッジの表示。
  - 自作クイズの分析画面において、各問題に対する誤答選択肢が何パーセントの割合で選ばれたかがグラフまたは表形式で正確に表示されること。
  - _Requirements: 9.1, 9.2, 9.3, 9.4_
  - _Depends: 2.1, 2.3, 2.4_

- [ ] 3.5 SNSワンクリック共有機能の統合
  - クイズ結果（スコア、解答時間）とクイズURLを含んだワンクリックシェアリンクの生成。
  - プレイ完了後に「共有」ボタンを押した際、結果のテキストと短縮URLを含むSNS投稿用のテキストが正しく生成されてプレビューされること。
  - _Requirements: 7.3_
  - _Depends: 3.3_

## 4. Validation: 総合テストおよびパフォーマンスチェック

- [ ] 4.1 高負荷シミュレーションおよびパフォーマンス検証
  - 100万PVの稼働目標に対する初期表示速度（0.5秒以下）と、高負荷スパイク時のエラー率（0.1%未満）のシミュレーション。
  - Lighthouse等の計測において、初期HTMLレスポンス速度が0.5秒以内、かつキャッシュ（ISR）が正しく適用されていることが確認されること。
  - _Requirements: 7.4, 7.5_
