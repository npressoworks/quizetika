# Requirements Document

## Introduction
クイズ一覧を表示する「検索画面」および「プロフィール画面（作成したクイズ）」において、最初は「もっと見る」ボタンをクリックすることで追加データが読み込まれ、それ以降はスクロール位置に応じて自動的にデータが追加ロードされる一貫したハイブリッド無限スクロールのUI/UXを提供します。また、プロフィール画面のデータロードはデータストアのカーソルを用いた20件ずつの段階的フェッチに変更され、不要な読み込みが削減されます。無料会員に対しては、ロードされたクイズカードの10件ごとにGoogle AdSenseのインライン広告が適切に挿入され続けます。

## Boundary Context
- **In scope**:
  - 検索画面における「もっと見る」ボタン ＋ 自動無限スクロール。
  - プロフィール画面における「もっと見る」ボタン ＋ 自動無限スクロール。
  - プロフィール画面でのデータストアのカーソルを使用したクイズの20件段階取得（検索ワード未入力時）。
  - クイズ一覧表示における無料会員向けの10件ごとの広告（`AdsenseInlineAd`）の適切な自動挿入。
- **Out of scope**:
  - トップ画面（カルーセル表示）の無限スクロール化。
  - マイクイズ画面（`/my-quiz`）の問題一覧テーブルの無限スクロール化。
- **Adjacent expectations**:
  - `quizeum-ads` (広告機能スペック) から提供される、ユーザーの有料プラン購読ステート（`showAds`）および `AdsenseInlineAd` 広告コンポーネントが正常に利用できること。
  - クイズのコレクションが適切に `createdAt` でソートされ、インデックスが機能していること。

## Requirements

### Requirement 1: 検索画面のハイブリッド無限スクロール
**Objective:** As a ユーザー, I want クイズ検索結果が段階的に追加表示されることで, so that ページのスクロールパフォーマンスを維持しつつ、素早く多くのクイズを見つけられること

#### Acceptance Criteria
1. When 検索結果ページが初期ロードされた時, the Search Page shall 最初は最大20件のクイズカードを表示し、かつ「もっと見る」ボタンを表示する。
2. While 自動無限スクロールモードが無効な時, when ユーザーが「もっと見る」ボタンをクリックした時, the Search Page shall 次のクイズ（最大20件）をロードして一覧に追加し、かつ「もっと見る」ボタンを非表示にし、かつ自動無限スクロールモードを有効にする。
3. While 自動無限スクロールモードが有効な時, when ユーザーがスクロールしてページ下部に到達した時, the Search Page shall 追加のクイズ（最大20件）を自動的にロードする。
4. If 追加できるクイズがこれ以上存在しない時, the Search Page shall 追加ロード処理を停止し、かつ「もっと見る」ボタンを表示しない。
5. While クイズの追加ロード処理が実行中の時, the Search Page shall スケルトン形式のプレースホルダー表示を一覧の下部に表示する。

### Requirement 2: プロフィール画面のクイズ段階取得とハイブリッド無限スクロール
**Objective:** As a プロフィール閲覧ユーザー, I want 作成したクイズが段階的に取得・表示されることで, so that データ転送量を節約しつつ、特定の作成者のクイズを多く見られること

#### Acceptance Criteria
1. While 検索入力欄（プロフィール内）が空の時, when プロフィール画面が初期ロードされた時, the Profile Page shall データストアから最新の20件のクイズをロードして表示し、かつ「もっと見る」ボタンを表示する。
2. While 検索入力欄（プロフィール内）が空で自動無限スクロールモードが無効な時, when ユーザーが「もっと見る」ボタンをクリックした時, the Profile Page shall データストアのカーソルを利用して次の20件のクイズをロードして一覧に追加し、かつ「もっと見る」ボタンを非表示にし、かつ自動無限スクロールモードを有効にする。
3. While 検索入力欄（プロフィール内）が空で自動無限スクロールモードが有効な時, when ユーザーがスクロールしてページ下部に到達した時, the Profile Page shall データストア의 カーソルを利用して自動的に次の20件のクイズをロードする。
4. If プロフィール内の検索入力欄に1文字以上の文字が入力された時, the Profile Page shall 作成者の全クイズ（または十分な上限数のクイズ）を一括ロードし、かつクライアントサイドでのフィルタリングおよび追加読み込み（「もっと見る」及び無限スクロール）のモードに切り替える。
5. If 作成者が自分自身であり、下書きも含む表示が有効な時, the Profile Page shall 公開中のクイズと下書きクイズを混在した状態で段階的ロード（または検索時一括ロード）を行う。

### Requirement 3: クイズ一覧への無料会員限定インライン広告の自動挿入
**Objective:** As a 無料会員, I want 読み込まれたクイズの間に適切に広告が差し込まれることで, so that レイアウトが崩れることなくコンテンツの間に自然なPR表示が行われること

#### Acceptance Criteria
1. Where ユーザーが無料会員（広告表示対象）である場合, the Quiz List Grid shall 表示されているクイズカードが10件増えるごとに、その間に1件の `AdsenseInlineAd` コンポーネントをインラインで自動挿入する。
2. Where ユーザーが有料会員である場合, the Quiz List Grid shall インライン広告を一切表示しない。
3. When 「もっと見る」や自動無限スクロールによってクイズカードが追加ロードされた時, the Quiz List Grid shall 追加されたカードを含めた全体のクイズカードのインデックス順に、10件ごとの位置に広告の挿入位置を再計算して表示する。
