# Requirements Document

## Project Description (Input)
### 目的と背景
ユーザー（クリエイター）がクイズを作成・編集する際、AIによるサムネイル生成だけでなく、自身のローカルデバイスから画像をアップロードしてカバー画像に設定する機能を提供します。

### 現状と課題
- 現在のクイズエディタ (`QuizMetadataSection`) には「AIでサムネイルを生成」ボタンと、生成されたプレビュー表示枠しか存在しない。
- Firebase Storage へのアップロードやルール (`storage.rules`) はすでに定義されているが、ローカルファイルの選択、およびそれを OGP 規格に整形してアップロードするフローがない。

### 達成したい状態
- クイズ作成・編集画面で、ユーザーがPC・スマホのローカルから画像（PNG, JPEG, GIF）を選択できる。
- 選択した画像を OGP 規格（1.91:1）のアスペクト比で直感的に切り抜けるトリミングモーダルが動作する。
- トリミング・切り出された画像は**最大解像度幅1920、高さ1005以下のJPEG形式**に変換され、サイズが10MB以下の状態で Firebase Storage の `quizzes/{quizId}/cover_{timestamp}.jpeg` に保存され、`thumbnailUrl` が更新される。

## Boundary Context
- **In scope**:
  - クイズエディタ (`QuizMetadataSection`) 内での画像ファイル選択UIの提供。
  - ファイル選択後の 1.91:1 固定アスペクト比によるトリミングUI（モーダル）の提供。
  - 切り出し時に最大解像度幅1920、高さ1005以下への縮小、および JPEG への形式変換の適用。
  - 変換・トリミングされた画像（JPEG、10MB以下）の Firebase Storage への遅延アップロード。
  - アップロードされた画像のダウンロードURLをクイズオブジェクトの `thumbnailUrl` に反映。
  - 既存のAIサムネイル生成機能との共存（排他的な上書き可能）。
- **Out of scope**:
  - 各設問（クイズ内の個別問題）での画像アップロード機能。
  - クイズ一覧以外の画像管理・クレンジング専用のユーザーUI（データ保存時のクレンジング等は既存処理を流用）。
  - アップロード後のサーバサイドでの画像変換・リサイズ処理（すべてクライアントサイドで完結）。
- **Adjacent expectations**:
  - クイズ保存処理（Firestore）が、設定された `thumbnailUrl` を既存ロジックのまま永続化できること。

## Requirements

### Requirement 1: クイズカバー画像のローカル選択とバリデーション
**Objective:** クリエイターとして、自作の画像をクイズのカバー画像に設定したいので、PCやスマホから画像ファイルを選択し、正しい形式・サイズであることを検証できるようにしたい。

#### Acceptance Criteria
1. When ユーザーが画像選択UIをクリックしたとき, the クイズエディタ shall デバイスのファイル選択ダイアログを表示する。
2. When ユーザーがファイルを選択したとき, the クイズエディタ shall 選択されたファイルのサイズと拡張子をチェックする。
3. If 選択されたファイルのサイズが10MBを超えるとき, then the クイズエディタ shall エラーメッセージを表示し、ファイルの読み込みを中断する。
4. If 選択されたファイルの拡張子が PNG, JPEG, GIF 以外（SVGを含む）のとき, then the クイズエディタ shall エラーメッセージを表示し、ファイルの読み込みを中断する。

### Requirement 2: 画像のトリミングとJPEG変換/サイズ調整UI
**Objective:** クリエイターとして、アップロードした画像がクイズカードに最適に表示されるようにしたいので、1.91:1 (OGP規格) のアスペクト比で直感的に切り抜け、自動的に上限サイズ以下のJPEGに最適化されるようにしたい。

#### Acceptance Criteria
1. When バリデーションを通過した画像ファイルが読み込まれたとき, the クイズエディタ shall トリミングモーダルを表示する。
2. While トリミングモーダルが表示されている間, the トリミングモーダル shall 画像を16:9の固定アスペクト比の切り抜き枠内に表示し、拡大・縮小および位置調整を行うUIを提供する。
3. When ユーザーがトリミング確定アクションを行ったとき, the トリミングモーダル shall 切り抜かれた画像データを最大解像度フルHD（1920x1080）に制限した上で JPEG 形式（Blob）で生成し、モーダルを閉じる。
4. When ユーザーがトリミングキャンセルアクションを行ったとき, the トリミングモーダル shall 画像の変更を反映せずにモーダルを閉じる。

### Requirement 3: トリミング画像の Firebase Storage へのアップロード
**Objective:** クリエイターとして、トリミングした画像をクラウドに保存したいので、Firebase Storage に自動アップロードされ、クイズのメタデータに反映されるようにしたい。

#### Acceptance Criteria
1. When トリミング画像JPEG Blobが確定したとき, the アップロードサービス shall Firebase Storage の `quizzes/{quizId}/cover_{timestamp}.jpeg` にデータをアップロードする。
2. While アップロード処理が実行されている間, the クイズエディタ shall カバー画像編集UI上でアップロード中を示すインジケータを表示し、関連するアクションボタンを無効化する。
3. When アップロードが正常に完了したとき, the クイズエディタ shall 取得されたダウンロードURLをクイズの `thumbnailUrl` に設定し、プレビュー画像を更新する。
4. If アップロード処理が失敗したとき, then the クイズエディタ shall アップロードエラーメッセージを表示し、以前の `thumbnailUrl` 状態を維持する。

### Requirement 4: AI生成サムネイルとの共存
**Objective:** クリエイターとして、手動アップロード画像とAI生成画像を柔軟に選択したいので、どちらの方法でもサムネイルを設定・上書きできるようにしたい。

#### Acceptance Criteria
1. When ユーザーがAIによるサムネイル生成に成功したとき, the クイズエディタ shall AI生成されたサムネイルURLを `thumbnailUrl` に設定する。
2. When ユーザーが手動で画像をアップロードしたとき, the クイズエディタ shall AI生成された画像を上書きし、アップロードされた画像のダウンロードURLを `thumbnailUrl` に設定する。
3. The クイズエディタ shall 既存の `thumbnailUrl`（AI生成か手動アップロードかを問わない）がクイズドキュメント内に存在する場合、それを初期プレビューとして表示する。
