# Implementation Plan - quizetika-ads

## 1. Foundation: 広告基盤のセットアップと共通フックの実装
- [x] 1.1 useAds フックの作成
  - ユーザーの認証状態および会員プラン情報を取得する共通ロジックをラップした React Hook を実装する
  - 有料会員（Stripeのサブスクリプションが有効なユーザー）の場合は、広告の非表示を判定するフラグをオフにする
  - クイズ完了時に 1/3 の確率で動画広告を表示するための確率判定処理を組み込む
  - テストおよびE2E用の LocalStorage モックキーによる割り込み制御（強制表示、強制非表示など）をサポートする
  - 単体テストを作成し、会員状態や LocalStorage の設定に応じて期待通りの判定値が返されることを検証する
  - **完了条件**: 単体テストスイートを実行し、広告表示可否および動画広告トリガーの全判定ロジックが正常にパスすること
  - _Requirements: 1.1, 1.2, 1.3, 2.3, 3.1, 3.5_

- [x] 1.2 AdSense 環境変数の定義と AdsenseProvider の実装
  - 環境変数ファイルに Google AdSense のパブリッシャーIDを定義する
  - 有料プラン以外の一般ユーザーに対してのみ、Google AdSense の外部スクリプトを動的に読み込むプロバイダーコンポーネントを実装する
  - ルートレイアウトにプロバイダーを組み込み、全体のロード制御を適用する
  - **完了条件**: 無料会員でアプリを開いた際には AdSense のスクリプトが読み込まれ、有料会員では一切スクリプトがマウントされないことをブラウザの開発者ツールで確認する
  - _Requirements: 1.1, 1.2_

## 2. Core: 主要広告コンポーネントの実装
- [x] 2.1 (P) AdsenseInlineAd コンポーネントの実装
  - クイズカードのグリッドに差し込まれるインライン広告用のコンポーネントを実装する
  - 広告表示対象外 of ユーザーには何も描画せず、レイアウトに隙間や崩れを作らないようにする
  - 本番環境では AdSense 広告をプッシュして表示し、開発・テスト環境ではダミーの「PR」スポンサー広告カードを表示する
  - 広告枠のカード上部に通常のクイズカードと識別可能な「PR」バッジを表示する
  - **完了条件**: 開発環境において、PRチップ付きのダミーのインライン広告カードが期待通りのカードスタイルで表示されること
  - _Requirements: 2.1, 2.2_
  - _Boundary: AdsenseInlineAd_

- [x] 2.2 (P) VideoAdModal コンポーネントの実装
  - クイズ完了から結果遷移までの割り込みを行うための、全画面動画広告モーダルを実装する
  - モーダル展開時に5秒間のカウントダウン処理を開始し、経過するまではスキップボタンを非活性化する
  - モーダル内にダミーの動画再生プレースホルダーと再生アニメーションを描画する
  - 5秒経過後に「スキップして結果へ」ボタンを活性化し、クリック時に完了コールバックを発火する
  - **完了条件**: モーダルが展開され、5秒経過するまでスキップできず、5秒後にスキップボタンが押せてモーダルが閉じられること
  - _Requirements: 3.2, 3.3, 3.4_
  - _Boundary: VideoAdModal_

## 3. Integration: クイズ一覧およびプレイフローへの広告統合
- [x] 3.1 クイズ一覧画面へのインライン広告差し込み
  - ホーム画面、検索結果、ジャンル別一覧、およびタグ別一覧のクイズカードグリッドのレンダリング部分を改修する
  - クイズ一覧が10件以上ある場合、10件ごとに `AdsenseInlineAd` コンポーネントを配列へ差し込む処理を追加する
  - ユーザーが有料会員である場合は、差し込み処理を行わず通常のクイズカードのみを表示する
  - **完了条件**: 無料会員で一覧画面をスクロールした際、10件目の直後に広告カードが挿入され、有料会員時には挿入されないこと
  - _Depends: 2.1_
  - _Requirements: 2.1, 2.3_

- [x] 3.2 クイズ完了遷移時の動画広告割り込み組み込み
  - 通常のクイズプレイ画面およびテストプレイ画面における、完了時の結果画面へのルーティング処理を修正する
  - クイズ完了時に動画広告の表示対象（確率 1/3）であるか判定し、対象の場合は結果画面遷移を一時停止して動画広告モーダルを起動する
  - モーダルがスキップ完了した際、退避させていた遷移先URLへの `router.push` を実行して遷移を完了させる
  - **完了条件**: 無料会員でクイズを完了した際、1/3の確率で動画広告モーダルが表示され、5秒後にスキップすると結果画面に正常に遷移すること
  - _Depends: 2.2_
  - _Requirements: 3.1, 3.2, 3.5_

## 4. Validation: 広告機能の Playwright E2E テストの実装
- [x] 4.1 E2E テストケースの追加
  - 新規に広告機能のE2Eテストファイルを作成し、テストケースを追加する
  - 無料プランユーザーにおいて、クイズ一覧へのインライン広告表示を検証する
  - 有料プランユーザーにおいて、広告スクリプトの未ロードおよび一覧への広告非表示を検証する
  - 動画広告の強制フラグを有効化した状態で、クイズ完了時にモーダルが表示され、5秒経過後にスキップして結果へ遷移できること、および有料プラン時は直接遷移することを検証する
  - **完了条件**: E2Eテストを実行し、作成した広告統合のすべてのテストシナリオが正常にパスすること
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.3, 3.1, 3.2, 3.3, 3.4, 3.5_
